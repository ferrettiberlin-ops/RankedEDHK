RankedEDHK — MVP local run & test

Quick goal
- Local MVP for internal testing: UI with left university box, program buttons, reviews list, and submit-review form.
- No full user signup: email validation only (one review per email).

Files added/updated
- `index.html`, `index.js`, `index.css` — simplified front-end layout and behavior
- `app/api/programs/route.ts` — GET `/api/programs?uni=...` (returns DB programs or fallback)
- `app/api/reviews/route.ts` — POST `/api/reviews` (anonymous submit, moderation, insert to Supabase or local fallback)
- `lib/supabase.ts` — client helpers
- `scraper-python.py` — scrapes JUPAS pages -> `data/programs_seed.json`
- `scripts/setup_db.js` — creates tables (via RPC) and seeds `program_list` from `data/programs_seed.json` or fallback
- `migrations/create_tables.sql` — SQL you can run in Supabase SQL editor
- `data/programs_seed.json` — generated by the scraper (contains ~365 programs)
- Fallback file on DB missing: `data/reviews_local.json`

Run locally (recommended)
1. Ensure `.env.local` contains your Supabase values (do NOT commit this file):

   NEXT_PUBLIC_SUPABASE_URL=https://<your-project>.supabase.co
   NEXT_PUBLIC_ANON_KEY=<anon-key>
   SUPABASE_SERVICE_ROLE_KEY=<service-role-key>
   HF_API_KEY=<huggingface-key>

2. Install (if needed) and run the scraper (creates `data/programs_seed.json`):

```bash
python3 scraper-python.py
```

3. Create tables and seed programs via the setup script (loads `.env.local` automatically):

```bash
node scripts/setup_db.js
```

If `scripts/setup_db.js` cannot create tables (some Supabase projects restrict RPC), run the SQL in `migrations/create_tables.sql` in the Supabase SQL editor, then run the seeding step manually by re-running `node scripts/setup_db.js`.

4. Start the Next.js dev server:

```bash
npm run dev
```

5. Open the app in the browser:

- http://localhost:3000 — select a University (left box), then click a Program (top bar) — program buttons are populated from Supabase `program_list`.
